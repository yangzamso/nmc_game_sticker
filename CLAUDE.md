# 작업 지침

## 사소한 수정은 빠르게

퍼센트/px/색상 같은 단순 수치 값 하나를 바꾸는 요청은 다음만 한다:

1. Edit으로 값만 수정
2. `git commit` + `git push`

**하지 않는다** (사용자가 명시적으로 요청하지 않는 한):
- `npm run dev` + Playwright 브라우저 실행 + 스크린샷 캡처
- GitHub PR 생성 + 병합

사용자가 "보여줘", "확인해줘", "캡처해봐" 라고 명시하거나, 값이 여러 곳에서 서로 연동되는 계산(예: 스테이지 비율이 바뀌면 캐릭터 위치 공식도 같이 바뀌는 경우)일 때만 브라우저로 실제 렌더링을 검증한다.

## PR/병합

- 수정할 때마다 PR을 만들지 않는다. 여러 커밋을 브랜치에 쌓아두고, 사용자가 "메인에 병합해줘"라고 말할 때 한 번에 PR 생성 → 병합한다.
- 병합 전에 매번 "병합할까요?"라고 되묻지 않는다 — 사용자가 이미 이 흐름(커밋/푸시 후 병합 요청)에 익숙하므로, 병합해달라는 요청을 받으면 바로 실행한다.

## 검증이 필요한 경우

- 여러 요소의 위치/크기가 서로 참조하는 계산식을 바꿀 때 (예: 로고 크기 변경이 캐릭터 위치 공식에 영향을 주는 경우)
- 레이아웃 구조 자체를 바꿀 때 (요소를 다른 부모로 옮기거나, 좌표 기준을 변경하는 경우)

이런 경우에만 Playwright로 실제 값을 측정하거나 스크린샷으로 확인한다. 이때도 관련 있는 화면 하나만 확인하고, 관련 없는 다른 화면(예: 코디 화면만 바꿨는데 인쇄 화면까지)은 캡처하지 않는다.

## 이 프로젝트 구조 메모

- 코디(옷 갈아입기) 스테이지: `src/components/GameBoard.jsx`, `GameBoard.module.css` — 스테이지는 4:5 비율, `svh` 대신 뷰포트 너비 기준으로 크기 고정 (카카오톡 인앱 브라우저 주소창 토글 대응)
- 인쇄 포토카드 생성: `src/utils/savePhotoCard.js` — 사진 영역은 항상 3:4 고정, 코디 스테이지 비율과 무관하게 분리되어 있음. 캐릭터는 로고 하단~사진 하단 사이 정중앙에 자동 배치(의상 높이 자동 반영)
- 인쇄 오버레이(프린터 애니메이션) UI: `src/components/PrintOverlay.jsx`
